# n7tya-lang 採用分析

## 概要

本ドキュメントは、n7tya-langを実環境で採用する際の判断材料を提供します。

---

## 移行する理由（メリット）

### 開発効率

| 理由 | 詳細 |
|------|------|
| **フルスタック統一** | Python + JS + 型安全を1言語で完結。チーム間の言語壁がなくなる |
| **AI/ML資産の活用** | PyTorch, LangChain, OpenAI SDKをそのまま使えて、かつ高速 |
| **Vibe-coding対応** | AIと人間の協調開発に最適化。ChatGPT/Copilotとの相性抜群 |

### パフォーマンス

| 理由 | 詳細 |
|------|------|
| **GCなし** | ストリーミング応答時のレイテンシが安定 |
| **Wasm対応** | ブラウザでもネイティブに近い速度 |
| **Pythonの100倍速** | 計算処理部分の大幅な高速化 |

### 開発体験（DX）

| 理由 | 詳細 |
|------|------|
| **ゼロコンフィグ** | 設定ファイルなしで即開発開始 |
| **ホットリロード** | 変更が即座にブラウザに反映 |
| **LSP標準搭載** | VSCodeで補完・エラー表示が最初から動く |
| **詳細なエラー** | miette式で原因と解決策を同時に提示 |

### セキュリティ

| 理由 | 詳細 |
|------|------|
| **デフォルト安全** | Deno式パーミッションで意図しないアクセスを防止 |
| **依存関係監査** | 脆弱性を自動検出 |

---

## 移行しない理由（リスク・障壁）

### エコシステム

| 理由 | 深刻度 | 対策案 |
|------|--------|--------|
| **ライブラリが少ない** | 高 | Python連携で補完、徐々に純正を増やす |
| **コミュニティが小さい** | 中 | Stack Overflowより公式Discord/GitHub活用 |
| **ドキュメント不足** | 中 | 公式ドキュメントとサンプルを充実させる |

### 組織・チーム

| 理由 | 深刻度 | 対策案 |
|------|--------|--------|
| **学習コスト** | 中 | Pythonライク構文で最小化、チュートリアル整備 |
| **チーム説得が困難** | 高 | 小さなPoCから始める、ROIを示す |
| **採用が難しい** | 中 | Python/TS経験者なら短期間で習得可能とアピール |

### 技術的リスク

| 理由 | 深刻度 | 対策案 |
|------|--------|--------|
| **本番実績なし** | 高 | 自社でdogfooding、OSS採用実績を作る |
| **デバッグツール未整備** | 中 | 初期段階でソースマップ、デバッガ対応を優先 |
| **言語が消える可能性** | 中 | 安定版まで本番投入を慎重に、企業スポンサー獲得 |
| **Python依存** | 低 | pyo3のオーバーヘッドは許容範囲内 |
| **破壊的変更** | 中 | 1.0リリースまではAPIが変わる可能性あり |

## 現実的な移行戦略

### Phase 1: 検証（1-3ヶ月）
1. 社内ツール/PoCで試用
2. AIチャットボットの小規模実装
3. パフォーマンス計測とPythonとの比較

### Phase 2: 部分採用（3-6ヶ月）
1. 新規の小規模プロジェクトで正式採用
2. 既存PythonプロジェクトのUI部分のみn7tya-langに
3. チーム内での知識共有・ドキュメント整備

### Phase 3: 本格採用（6ヶ月以降）
1. 新規プロジェクトはn7tya-lang優先
2. 既存プロジェクトの段階的移行
3. 社内標準言語として推進

## 採用判断チェックリスト

### 採用を推奨するケース
- [ ] AI-Agent Webアプリを新規開発する
- [ ] Python + JS の二重開発に疲弊している
- [ ] パフォーマンスが重要（リアルタイム処理など）
- [ ] Vibe-codingを積極的に活用したい
- [ ] チームが新技術への挑戦に前向き

### 採用を見送るべきケース
- [ ] 安定性・実績が最優先（金融、医療など）
- [ ] 大規模チームで一斉移行が必要
- [ ] 特殊なライブラリがPython連携で動かない
- [ ] 社内にRust経験者がおらず、処理系への貢献が困難
